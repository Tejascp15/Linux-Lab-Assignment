Question 3 – File System and Link Analysis

1. Original File Creation

Command:
echo "Linux Lab" > file.txt

Observation:
A text file named file.txt was created containing the text "Linux Lab". The file was verified successfully using the cat command.


2. Hard Link Creation

Command:
ln file.txt hardlink.txt

Observation:
A hard link named hardlink.txt was created successfully. It points directly to the same inode as the original file.

3. Symbolic Link Creation

Command:
ln -s file.txt symlink.txt

Observation:
A symbolic link named symlink.txt was created successfully. It stores the pathname of the original file rather than sharing its inode.

4. Inode Analysis

Command:
ls -li

Observation:
The original file (file.txt) and hardlink.txt have the same inode number, showing that they refer to the same file. The symbolic link has a different inode because it is a separate file containing the path to the original file.


5. Metadata Analysis

Command:
stat file.txt
stat hardlink.txt
stat symlink.txt

Observation:
The metadata confirmed that file.txt and hardlink.txt shared the same inode number, while symlink.txt had a different inode and referenced the original file.


6. Effect of Deleting the Original File

Command:
rm file.txt

Observation:
After deleting file.txt, hardlink.txt continued to work because it still referenced the file's inode. The symbolic link became broken because its target file no longer existed.



Conclusion

Hard links share the same inode as the original file and remain functional even after the original filename is deleted. Symbolic links store the pathname of the original file and become invalid if the original file is removed.
