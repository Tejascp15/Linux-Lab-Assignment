Question 4 – File Access and I/O Investigation

1. Open Files

Command:
lsof

Output:
bash: lsof: command not found

Observation:
The lsof utility is not available in the Coursera Lab environment, and additional package installation is not permitted. Therefore, the command could not be executed.


2. File Descriptors

Command:
ls -l /proc/$$/fd

Observation:
The command displayed the file descriptors of the current shell. It showed the standard input (0), standard output (1), standard error (2), and other open descriptors.


3. Output Redirection

Commands:
echo "Hello Linux" > output.txt
cat output.txt

Observation:
The > operator redirected the command output to output.txt. The file was successfully created and contained the text "Hello Linux".


4. Error Redirection

Commands:
ls abc 2> error.txt
cat error.txt

Observation:
The 2> operator redirected the error message generated by the ls command into error.txt instead of displaying it on the terminal.


5. Redirecting Standard Output and Standard Error

Commands:
ls > out.txt 2> err.txt
cat out.txt
cat err.txt

Observation:
The directory listing was successfully stored in out.txt. Since no error occurred, err.txt remained empty.


6. Resource Limits

Command:
ulimit -a

Observation:
The ulimit command displayed the current resource limits for the user, including limits on open files, memory, stack size, and processes.


Conclusion

Linux manages file input and output using file descriptors. Standard output and standard error can be redirected independently, making it easier to store program output and error messages in separate files for debugging and logging.
